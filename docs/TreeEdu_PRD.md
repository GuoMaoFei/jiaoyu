# 智树 (TreeEdu) Agent - 产品架构与需求文档 (PRD)

## 1. 产品概述
智树 (TreeEdu) Agent 是一款基于 `VectifyAI/PageIndex`（一种无向量、基于树状推理的 RAG 技术）构建的智能教育应用。本产品旨在解决传统教育AI中易出现的“幻觉”、“超纲”和“脱离学情”等问题。通过深度解析教材目录树和知识点结构，智树能够提供完全依附于教学大纲的个性化伴学与教务管理功能。

## 2. 核心价值主张 (Core Value Proposition)
* **对于学生**：一个永远不会超纲、能够根据个人吸收能力和情绪状态动态调整进度的 24 小时专属苏格拉底式私教。
* **对于教师/教务**：一个智能的学情分析大脑和高质量的变式出题引擎，大幅降低阅卷、组卷和辅导负担。
* **技术壁垒**：利用 PageIndex 的 Tree Search 能力，使 AI 的推理和检索（Retrieval）具有与人类专家一致的逻辑树层级，确保知识点定位的 100% 准确性。

## 3. 架构设计方法论 (Architectural Design Methodology)

### 3.1 教学理念融合
我们的系统架构不仅仅是技术的堆叠，更是现代教育心理学的数字化实现：
1. **维果茨基“最近发展区”理论 (ZPD)**：通过动态学情分析（正确率、耗时），Agent 每次推荐的题目难度或讲解深度，都刚好处于学生“跳一跳够得着”的区间。
2. **斯金纳“强化理论”**：结合“情绪与鼓励引擎”，在学生受挫时给予正向的情感强化，在学生连续答对时给予升维挑战。
3. **布鲁纳“发现学习法”**：Agent 默认采用苏格拉底式提问，引导学生主动探索结构化的知识树，而非被动接收答案。

### 3.2 顶层功能架构 (Macro Architecture)

整个系统分为三大平台、五大核心引擎：

**V1.0 Web 响应式架构与学习微生态 (Responsive Web Learning Ecosystem)**：
初期产品主要面向 C 端家庭场景，依托 AI 提供极简的浏览器端（PC/Pad/Mobile H5）智能服务。放弃重型原生 App，通过 PWA/Web 技术实现全平台覆盖：
1. **云端知识中枢 (Cloud Knowledge Hub)**：由平台官方维护的基础库，利用 PageIndex 预置海量主流教材的树状结构，无需人工干预。
2. **Student Portal (学生端 - 响应式 Web)**：核心交互入口。适配大屏脑图沉浸学习与手机端碎片化问答。支持调用设备摄像头（Pad/手机）直接拍照上传错题。功能包括：**按教材规划的课程学习**、智能计划生成、习题演练、拍题精讲、错题本与知识树可视化。
3. **Parent Portal (家长端 - 共用 Web 平台)**：主打"无压力监督与陪伴"。家长通过同一 Web 系统登录，以独立角色视图查看 AI 生成的柔性学情周报、情绪状态摘要及知识健康度概览，无需单独的 App 或第三方平台依赖。
4. **Core Engine Server (AI 核心计算层)**：基于大模型和 PageIndex 的树状搜索与多模态推理底层服务。保留了完整的历史会话以支持跨端（如手机切到 PC）的连续学习记录。

**五大核心引擎与结构化记忆库 (Core Engines & Structural Memory)**：
1. **Tree-Index Engine (树状索引双树引擎)**：负责解析 PDF 教材/题库，提取结构化知识树。为解决图文排版导致的提取歧义，采用 **Dual-Tree Mapping (双树语义映射)** 技术：前端展示由视觉大模型 (VLM) 提取的真实人类排版目录，底层与 PageIndex 生成的散乱知识树节点进行 LLM 语义软挂载映射，做到展示精确、检索强大。
2. **Structural Memory Overlay (结构化记忆覆写层) [核心新增]**：彻底抛弃强制大模型记住几个月闲聊记录导致“Token 爆炸”的传统做法。系统提取评估结果存入个人的“知识掌握表”（`health_score` 和历史错因），在每次检索 PageIndex 标准树时，动态将此表覆写（Overlay）到提纲节点上。大模型看到的永远是最冷酷、最精准的学情结构数据。
3. **Dynamic Routing Engine (动态规划引擎)**：结合基准骨架和学情数据，实时生成和调整个人学习计划。
4. **Socratic QA Engine (苏格拉底问答引擎)**：负责与学生的启发式对话，严格限制在教材关联节点内作答。
5. **Variant Generation Engine (变式生成引擎)**：根据错题和选定知识点，基于教材风格生成难度可控的新题。
6. **Empathy Engine (共情与鼓励引擎)**：监控交互状态（深夜、高频错误），触发心理安抚机制。

## 4. 详细需求清单 (Feature Requirements)

### 4.1. 云端知识中枢与家长配套服务 (Cloud Hub & Parent Services)
不再依赖繁重的教师管理后台，所有基础设施走向“自动化”和“轻量化陪伴”。

| 模块 | 功能名 | 优先级 | 功能描述与验收标准 |
| :--- | :--- | :---: | :--- |
| **基础资源** | **官方教材知识树预置** | P0 | **描述**：平台提前通过 PageIndex 跑通北师大版、人教版等主流教材，形成即开即用的官方知识树池。<br>**验收**：学生注册后可直接搜索并添加其正在使用的教材版本。 |
| **家长配套** | **自传资源 (自助教辅)** | P1 | **描述**：若官方资源库缺少某特定练习册，支持**家长/学生**直接拍照或上传 PDF。<br>**验收**：引擎自动后台解析，将其挂载到学生的私人库中辅助 Agent 答疑。 |
| **家长配套** | **AI 柔性家长周报 (Web 端)** | P0 | **描述**：AI 每周自动生成报告，家长通过 Web 端"消息中心"页面登录查阅孩子的薄弱点与高光时刻。系统支持站内通知提醒和邮件提醒。<br>**验收**：语言必须转化为"人类能听懂的鼓励式话术"（如："您的孩子这周在解析几何上遇到了些挑战，但TA坚持问了5个好问题，请多鼓励TA！"），取代冷冰冰的分数排名。 |

### 4.2. 学生使用端 (Student Portal) - 自主驱动引擎

这是产品的绝对核心。所有依赖教师的“派题、组卷、定计划”动作，全部由内建 Agent 自动化完成。

| 模块 | 功能名 | 优先级 | 功能描述与验收标准 |
| :--- | :--- | :---: | :--- |
| **辅助学习** | **按教材课程大纲学习 (Guided Learning)** | P0 | **描述**：系统将教材知识树自动转化为可导航的课程大纲列表(章-节-知识点)。学生选定一个知识节点后进入学习模式，Agent 按照"导入关联-新概念讲解-例题演示-即时巩固-小结解锁"五步法引导学习。Agent 严格引用教材原文进行讲解，不超纲。学习过程支持中断恢复(切设备/退出后从上次讲到的地方接续)。<br>**验收**：学生点击某课程节点并选择"开始学习"后，Agent 从教材中提取该节点内容分步讲解。讲解中穿插理解确认题。学完后节点自动解锁(`is_unlocked=true`)，下一课程节点可见。 |
| **辅助学习** | **对话式学情评估 (Conversational Assessment)** | P0 | **描述**：Agent 在对话和辅助学习中主动提问即构成"隐式评估"--学生在不知不觉中被评估，无考试焦虑。每次学生回答 Agent 的提问，系统自动生成轻量级评估记录并实时更新对应知识节点的 `health_score`。<br>**验收**：Agent 讲解中每提出一道理解检查题，后台自动写入一条 `CHAT_ASSESSMENT` 记录，关联到具体知识节点。学生无需做"正式测试"也能看到知识树健康度变化。 |
| **学情诊断** | **入学全量诊断 (Entry Diagnostic)** | P0 | **描述**：学生首次激活教材后，系统提供"全量摸底"机会。Agent 从每个章节各抽 1-2 道代表性题目组成诊断卷。做完后批量更新节点的 `is_unlocked` 和 `health_score`，已掌握章节直接跳过，薄弱章节标记重点。<br>**验收**：学生可选"快速摸底"或"从头开始学"。摸底后知识树自动着色，已掌握节点跳级解锁。 |
| **自主规划** | **AI 握手与基准计划生成** | P0 | **描述**：系统不再有"教务预设要求"。学生选定教材后，Agent 会主动询问考期目标和每日可用时长。<br>**验收**：Agent 在底层生成带有明确动作语义 (`task_type`如学习、做题、复习) 的每日学习任务清单。 |
| **自主规划** | **自适应动态路线 (Adaptive Routing)** | P0 | **描述**：计划是活的。如果学生某天状态极差、正确率低，Agent 不会像老师那样强制推送大量惩罚性作业。<br>**验收**：AI 自动延长该卡点知识的停留期，并在第二天更换更通俗的话术或材料重新讲解。前端需清晰展示任务卡片的类型变更。 |
| **智能问答** | **跨端连贯对话与拍题精讲** | P0 | **描述**：学生可通过 Pad/手机摄像头上传学校作业或错题，或在 PC 端截图。聊天状态云端同步。系统后台统一隔离保护用户上传的图像资源（防盗链）。<br>**验收**：Agent **先引用已选定教材的对应原句**进行启发式提问，绝不直接给最终答案。如果在 Pad 上答一半换到 PC 上，Agent 能接续之前的指导。 |
| **系统出题** | **Agent 自主智能组卷(严守边界与快照保护)** | P0 | **描述**：原“教师一键组卷”功能下放。支持局部知识点加练与全局大考组卷。<br>**验收**：Agent 组合一套定制卷子。**系统底层强制校验节点解锁状态 (`is_unlocked`)**，确保试卷绝对不超纲。卷面题源基于官方库做换皮生成，且试卷生成瞬间做物理快照备份，避免污染官方库与日后题库变动导致考卷丢失。 |
| **复习机制** | **艾宾浩斯靶向复习 (真实攻克拦截)** | P1 | **描述**：无需老师一键打包共性错题。复习引擎精确到具体的历史错题。为防“假阳性”，错题必须“连续答对 N 次”才算真正掌握。<br>**验收**：Agent 检索到到期的错题，自然地插入诸如“今天学新课前，我们先做一道你上周做错的变式题好吗？” |
| **自主检测** | **知识点微测 (Node-Level Micro-Test)** | P1 | **描述**：当学生学完一个极其微小的模块（如某一片树叶），想验证自己是否有资格获得“绿色状态”时。<br>**验收**：可以直接点击知识树上的该节点，主动要求 Agent：“请给我出 3 道题考考我”。 |
| **自主检测** | **阶段大考拉练 (Cumulative Global Exam)** | P1 | **描述**：当学生感觉本月学得很扎实，想要一次期中级别的模拟测验时。<br>**验收**：学生可以在首页主动发起"阶段大考"，Agent 会扫描系统内所有当前教材 `is_unlocked=true` （已学过）的节点，综合配出一套符合正规考试题型分布（如 10道选择+5道填空+3道大题）的综合卷。 |
| **自主检测** | **阶段全量复盘 (Periodic Full Review)** | P1 | **描述**：每月或学期末，系统主动建议一次"全量复盘"--与阶段大考不同，全量复盘从**所有已学节点**均匀抽题，目的是重新校准节点 `health_score`，避免长期未触及的节点数据过时。<br>**验收**：系统展示"已评估覆盖率"，当覆盖率低于 50% 时主动建议复盘。 |
| **个人中心** | **响应式知识树图谱** | P1 | **描述**：用养成游戏的逻辑替代考分排名。<br>**验收**：可视化的 2D 树图谱（PC端自适应全屏展现，Mobile端支持手势缩放折叠）。已掌握节点变绿，薄弱节点枯黄。 |

## 5. 核心交互流程设计 (Core Interaction Flows)

在 V1.0 C 端场景下，为了保证“极简、陪伴、有效”的原则，我们设计了以下五大核心场景的时序与流转逻辑：

### 5.1 场景一：首次破冰与智能计划生成 (Onboarding & Planning)
此流程的核心是在没有老师布置任务的情况下，系统如何“拉拽”学生进入学习状态。

1. **选择教材**：学生登录，在"云端知识中枢"搜索并添加当前使用的教材（如：人教版八年级数学上册）。系统在后台静默加载该教材的 PageIndex 树状索引。
2. **目标探询 (AI 握手)**：Agent 发起对话："Hi, 我是智树。这学期你想达到什么目标呢？每天大约能抽多少时间和我一起学习？"
3. **入学路径选择**：Agent 提供两条路径：
   * **路径 A：全量摸底诊断** - Agent 从教材知识树的每个章节各抽 1-2 道代表性题目，组成一份覆盖全书的诊断卷。做完后系统批量更新所有被覆盖节点的 `is_unlocked` 和 `health_score`，已掌握的章节直接跳级解锁，薄弱章节标记为重点学习区。
   * **路径 B：从零开始学** - 适合对教材完全陌生的学生，从第一章第一节按顺序开始 Agent 引导式学习。
4. **生成日程表 (Routing)**：Core Engine 根据 (未解锁的教材节点数 ÷ 剩余天数) 结合每天的 (承诺可用时长)，倒推出每日的学习清单（含"学新课"和"巩固复习"两类任务），并在日历视图中呈现。
5. **家长端同步**：家长登录 Web 端后，在消息中心收到通知："您的孩子已制定了《八上数学》的学习计划，预计每天投入 30 分钟。让我们一起陪伴TA！"”

### 5.2 场景二：Agent 引导式课程学习 (Guided Learning Session)
这是产品的学习核心——学生如何在 Agent 引导下学习新知识。Agent 不是丢 PDF，而是主导结构化讲解。

1. **选择课程节点**：学生在课程大纲列表中点击一个可学习的知识节点（如"2.2 轴对称的性质"），点击"开始学习"进入学习模式。
2. **导入与关联 (Step 1)**：Agent 先回顾关联知识——"上节课我们学了轴对称的概念，你还记得什么是对称轴吗？" 通过回忆激活上节课内容（同时产生一条对话评估记录）。
3. **新概念讲解 (Step 2)**：Agent 引用教材原文/图片，分段讲解本节核心概念。每讲完一小段，插入 1-2 道理解确认小问题。学生可随时打断提问，Agent 暂停讲解回答后继续。
4. **例题演示 (Step 3)**：Agent 抽取教材中的典型例题，不直接给答案，而是引导学生一步步推导——"已知△ABC中，AB=AC，你觉得第一步该怎么做？"
5. **即时巩固练习 (Step 4)**：Agent 出 2-3 道针对本节的练习题。每道题的作答结果自动写入 `CHAT_ASSESSMENT`，实时更新该节点的 `health_score`。
6. **小结与解锁 (Step 5)**：Agent 总结本节要点——"你已经掌握了轴对称性质的核心内容！下一节'等腰三角形'已解锁，想继续吗？" 系统更新 `is_unlocked=true` 并设置初始 `health_score`。
7. **中断恢复**：学习过程中途退出后，`LESSON_PROGRESS.current_step` 记录当前讲到的步骤。再次进入时 Agent 从断点接续——"上次我们讲到了轴对称性质的第二个定理，还记得吗？"

### 5.3 场景三：苏格拉底式跨端辅导与拍题精讲 (Web/Pad Socratic Tutoring)
这是体现 PageIndex 无向量树状推理优势的最核心流程，确保 AI 绝对不超纲、记录不丢失。

1. **学生发问/拍题 (Pad/Mobile/PC)**：学生在平板上调用摄像头拍下一张错题并发射："这题我不会"。
2. **多模态解析**：系统提取图片信息，持久化至 `CHAT_MESSAGE` 表。识别考点。
3. **知识树检索 (Tree-Search)**：Core Engine 调用 PageIndex。大模型**不基于公网知识**作答，而是沿着当前教材目录树，向下检索并**记录推理路径**。
4. **启发式对话 (Socratic Loop)**：
   * **Agent 引用出处**："这道题考察的是完全平方公式，这在我们教材的第 28 页有讲到。你还记得公式展开后中间那一项是什么吗？"
   * **学生作答**——系统自动生成 `CHAT_ASSESSMENT` 记录，实时评估学生对该知识点的掌握程度。
   * **Agent 引导与拆解**：如果学生答错，Agent 截取教材对应原文或公式图进行再次引导，直到学生自己一步步推导出正确结果。
5. **学情更新**：对话结束后，该题被自动收入错题本。对应知识树上的该节点"健康度"下降（变黄）。

### 5.4 场景四：日周期的自适应与靶向复习 (Adaptive Review Loop)
取代了传统的统一周测，系统利用碎片化时间，精准打击到单题级别的错题。

1. **每日登录问候**：学生次日通过 Web 或平板打开系统。Agent 拉取历史会话记录判定状态。
2. **靶向错题打捞 (Ebbinghaus Inject)**：Agent 扫描 `STUDENT_MISTAKE` 表，发现某一道特定的"全等三角形"错题达到了复习临界日（`next_review_date`）。
3. **变式前戏**：Agent 拦截今日的新课任务，通过 Variant Engine 根据当年的该错题生成一道变式题："今天的新课前，看看你能否搞定这道老同学的'变形'？"
4. **状态评估 (Empathy Engine)**：
   * **做对**：节点变绿，对应错题状态解除。解锁今日新课卡片。
   * **做错**：如果耗时过长，Empathy Engine 介入安抚，并动态修改数据库中今日 `PLAN_ITEM` 的属性（将 LEARN_NEW 改为 REVIEW）。
5. **家长周报 (Web 端推送)**：周末，Agent 汇总知识树变化和每周情绪总评，生成 Markdown 报告存入 `PARENT_WEEKLY_REPORT`，家长登录 Web 端消息中心即可阅读。系统同时发送站内通知和邮件提醒。

### 5.5 场景五：主动查漏补缺与多维自主成卷 (Self-Assessment & Test Gen)
针对学生最需要的“考前/周末巩固”和“局部自我证明”，系统完全开放试卷的定义权，并支持不同维度的自我拉练。

1. **测验发起 (多维度)**：
   * **维度 A (微观验证)**：学生在看完新课后，点击单独某个知识叶子节点，选择**“巩固此节点”**。Agent 立马下发 3~5 道只针对此考点的小测验。
   * **维度 B (宏观拉练)**：周末或考前，学生点击**“阶段自测”**。这会涉及全局统筹。
2. **严苛且安全的学情体检 (Strict Health Check)**（针对阶段自测）：
   * Agent 扫描该学生本周知识树。找准盲区：重点筛选出所有 `health_score < 80` 的薄弱节点。
   * **控制硬边界**：强行排除掉该学生数据库中 `is_unlocked = false` （未学习）的节点，**确保绝对不超纲**。
3. **Agent 自主命题组卷 (Test Assembly & Snapshot Validation)**：
   * **找真题做骨架**：Agent 利用 PageIndex 检索官方库（`source_type = OFFICIAL`）抽取原题。
   * **变式引擎与快照锁死**：Agent 使用大模型进行换皮造题。为了隔离不可控力（如日后官方题库修改），Agent 在组装这套专属试卷时，会将最终的 Markdown 题干全部写死在试卷记录表 (`snapshot_question_md`) 的快照中，确保历史考试永不因外部变异而崩溃。
4. **即做即判与反哺 (Streak Validation)**：
   * 批改完成后，做对的昔日薄弱错题，其连续正确计数器 (`consecutive_correct_count`) 加 1。只有当该计数器满足阈值（如连续2次做对不同变式），该错题状态才转变为 `MASTERED` (彻底攻克)，杜绝碰运气造成的假阳性。
   * 全新的薄弱错题依然触发场景二的苏格拉底错题拆解。此时这套动态卷本身也化作了学生的“新错题库沉淀”。

## 6. 后续设计演进方向 (V2.0 B2B2C 升级计划)

在完成 V1.0 的 C 端家庭自主学习闭环后，为了拓展商业生态和下沉至公立/机构教学场景，系统将演进出配套的 **B 端平台 (B2B2C 模型)**。在这一阶段，引入并赋能“教师”与“教务”角色：

### 6.1 教师工作台 (Teacher Workspace)
专为一线教师减负增效的配套生产力工具，数据完全与学生端的知识树互通。

* **私有域教辅挂载池**：老师可一键上传本班独有的练习卷/讲义 PDF。系统后台解析后，生成“私域知识树分叉”，Agent 只针对该老师所带班级的学生开放这些私有题库的检索。
* **群像学情大屏 (Class Radar)**：以班级为单位，将几十名学生的数据融合。知识树上标红的节点即为“本班共性薄弱区”，辅助教师调整实际课堂进度，实现精准教研。
* **一键靶向变式派发**：教师选中树上本周错误率最高的一个知识点，指令 Agent：“以该知识点为核心，参考昨天的卷子难易度，出一套 10 题的随堂测。”任务可定向推送到学生的 Student Portal 中。

### 6.2 机构/教务后台 (Admin Portal)
满足学校、培训机构管理层的高维权限和数字资产沉淀。

* **校本题库资源管理**：统筹管理机构自行研发的各类核心教材，设定全校基础课时大纲。
* **班师生拓扑关系链 (RBAC)**：维护“学校-年级-班级-教师-学生-家长”复杂的多层级权限与数据隔离矩阵。
* **学年质量监控**：追踪对比平行班级对同一本教材/知识树的掌握深度差异，辅助教学管理考核。

### 6.3 其他体验侧延展
* **激励体系 (Gamification)**：增加“浇水值”、“探索者徽章”等轻量化游戏化元素，进一步激发 C 端用户的内驱力。
* **数字人伴学 (Virtual Avatar)**：前端接入实时语音与数字人驱动，让“智树”拥有更温暖的形象。
* **自主学伴匹配 (Peer Learning)**：在完全匿名的前提下保护隐私，基于知识树卡点，为其随机匹配一名有相同错题经历的“同等水平神秘学伴”的解答历程。
